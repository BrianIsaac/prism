# Prism v2 — Claude Code instructions

Prism is an agentic city-discovery engine for the GrabMaps API Hackathon. Three LLM
agents (Claude Opus 4.7, OpenAI GPT 5.5, Google Gemini 3.1 Pro) share one harness,
one prompt, and one GrabMaps tool belt — the only differentiator is the model.
Their plans are scored by a frozen harness, HITL-rated by the operator, and pinned
to a shared MapLibre canvas.

The hackathon build is sharded across `docs/hackathon-phases/`. Start there.

## Coding standards

### Backend (`backend/`)

- Python 3.12, FastAPI, SSE streaming via `sse-starlette`
- `uv` for dependency management. Never use `pip` or `pip-tools`
- Google-style docstrings on every function, module, and class
- Full type hints on every function signature (parameters and return)
- `async def` for anything that touches I/O
- British English in prose (docstrings, log messages, commit bodies)
- No emojis in source, logs, or commits
- `from __future__ import annotations` at the top of every module so forward references resolve

### Frontend (`frontend/`)

- Next.js 15 App Router, React 19, TypeScript strict
- Tailwind CSS with shadcn primitives in `components/ui/`
- Invoke these skills on every component you write:
  - `vercel-react-best-practices`
  - `vercel-composition-patterns`
  - `shadcn` — for adding or updating primitives
  - `web-design-guidelines` — accessibility + UX checks
- Named exports (`export function Foo()`), kebab-case file names, `@/` import alias
- British English in copy and comments
- No emojis

## Live-only

Prism v2 has no mocks. There is no `USE_MOCK_GRABMAPS` toggle, no fixture JSON files,
and no synthetic local tools. Every value the agents see is produced by a real
GrabMaps endpoint or a real composite of real calls. If a live call is unavailable,
fail loudly rather than substituting fake data.
